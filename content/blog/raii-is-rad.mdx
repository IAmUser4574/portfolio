Considerable compiler, runtime, and application code is designed around memory safety and most software environments reside in one of two camps: You're either manually tracking and releasing memory 
(e.g. C/C++) or automatically managing memory through runtime garbage collection (e.g. Java, C#, Python).

The problem with manual is human overhead; it's tedious and error-prone for developers to constantly check out and then restock the toys they play with.

And problem with automatic is compute overhead and non-determinism  - automatic garbage collection in Java, C#, Python, Javascript is expensive and can run at undesirable moments in your code.
